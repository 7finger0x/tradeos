# Implementation Plan: Unified Platform Foundation

**Branch**: `001-platform-foundation` | **Date**: 2026-06-28 | **Spec**: [spec.md](./spec.md)

## Summary

Establish the shared technical foundation for Track A (AI Trading OS) and Track B (Hermes
Protocol): monorepo structure, Supabase schema with RLS, RBAC model, audit logging, auth-
integrated dashboard shell, and CI pipeline gates.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20+

**Primary Dependencies**: Next.js 15, React 19, Tailwind CSS, shadcn/ui, Supabase JS, Zod

**Storage**: Supabase Postgres (primary), Supabase Storage (screenshots/imports)

**Testing**: Vitest (unit), Supabase pgTAP/SQL tests (RLS), Playwright (E2E later)

**Target Platform**: Web (Vercel/Render), Linux CI

**Project Type**: Modular monorepo (web + API + workers + shared packages)

**Performance Goals**: Dashboard TTFB < 1s staging; auth check < 200ms p95

**Constraints**: Fail-closed security; no live execution; broker read-only; tenant_id required

**Scale/Scope**: Phase 1 foundation only—no journal import, briefing, or Hermes adapters yet
